# AGS Tools User Guidance

This documentation is split into smaller guides so that non-coders and geotechnical engineers can find the right starting point quickly.

## Start Here

If you are new to QGIS, read this first:
- [QGIS Basics for Geotechnical Engineers](QGIS_BASICS_FOR_GEOTECH_ENGINEERS.md)

If you want to understand which plugins and tools support AGS visualisation and export, read this next:
- [AGS Visualisation and Export Tools](AGS_VISUALISATION_AND_EXPORT_TOOLS.md)

## Recommended Reading Order

1. Learn the basic QGIS concepts: projects, layers, attribute tables, styling, and plugin access.
2. Review the AGS-related tools and plugins available in this repo and externally.
3. Use the detailed technical docs in this repo when you need implementation detail.

## Other Useful Documents in This Repo

- [README.md](README.md) for the repo overview and installation context
- [DATA_FLOW.md](DATA_FLOW.md) for the AGS to DB to CSV workflow
- [DOCUMENTATION - work in progress.md](DOCUMENTATION%20-%20work%20in%20progress.md) for more technical detail

## Intended Audience

These guides are written for:
- geotechnical engineers
- engineering consultants
- GIS users
- project teams who want to use QGIS without writing code

## Scope

These guides focus on:
- understanding QGIS from an engineering user perspective
- understanding which plugins help with AGS visualisation
- understanding where this repo fits into a wider AGS to GIS to reporting workflow

They do not try to teach Python, plugin development, or the internal codebase in depth.
